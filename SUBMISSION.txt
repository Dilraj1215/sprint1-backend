TECHNICAL QUESTIONS — PREPARATION
================================================================================

Q: Where are your API routes defined?
A: In /routes/ — authRoutes.js, taskRoutes.js, userRoutes.js, categoryRoutes.js.
   Imported and mounted in server.js under /api/*.

Q: How does the frontend connect to the backend?
A: frontend/src/api/client.js has a fetch-based request() helper that
   attaches the Bearer token from localStorage. All screens import from
   this file. In development the Vite proxy forwards /api to port 3000;
   in production Express serves both on the same origin.

Q: How is authentication handled?
A: POST /api/auth/login returns a JWT. The token is stored in localStorage
   and sent as an Authorization: Bearer header on every protected request.
   The protect middleware in middleware/authMiddleware.js verifies the token
   and attaches req.user so controllers know which user is acting.

Q: How is user data scoped?
A: The task controller reads req.user.id (from the decoded JWT) instead of
   trusting the request body. Task.findByUserId(), Task.findByUserAndStatus(),
   and Task.getStatisticsByUser() all filter with WHERE user_id = $1.

Q: How is the frontend deployed alongside the backend?
A: npm run build (in the root) runs: cd frontend && npm install && npm run build
   This creates frontend/dist/. server.js uses express.static to serve those
   files. A wildcard GET * route sends index.html so React Router works on
   page refresh.
